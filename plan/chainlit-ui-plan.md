# JobOracle Chainlit 临时前端规划

## 1. 目标

这一版 `Chainlit` 前端的目标不是做最终产品，而是：

1. 验证多轮对话是否顺畅
2. 验证记忆是否正确累积
3. 验证“聊天回复”和“生成报告”两种模式是否切换自然
4. 在临时前端阶段，尽量做到不简陋、易理解、好看

所以这版前端的核心关键词是：

- 轻量
- 清晰
- 美观
- 易调试

## 2. 总体布局

建议页面拆成 4 个主要部分：

1. 顶部品牌区
2. 主对话区
3. 右侧状态面板
4. 底部输入区与快捷操作区

如果第一版 `Chainlit` 布局受限，至少也要保证：

- 聊天流在中间
- 侧边有状态信息
- 底部有用户输入框
- 底部或侧边有关键按钮

## 3. 页面结构

### 3.1 顶部品牌区

作用：

- 建立产品识别
- 让页面第一眼不显得像默认 demo
- 提醒用户当前模式是“求职咨询助手”

建议展示内容：

- Logo
- 产品名 `JobOracle`
- 一句简短副标题

建议文案：

- 标题：`JobOracle`
- 副标题：`多轮求职分析与就业决策助手`

视觉建议：

- 标题偏大，字重更高
- 副标题用较浅颜色
- 顶部可有一条很轻的渐变分隔

## 4. 主对话区

这是页面最重要的区域。

作用：

- 显示多轮消息
- 承担主要用户交互
- 展示系统状态变化

建议消息类型：

1. 用户消息
2. 助手回复
3. 系统状态消息
4. 报告生成结果卡片

### 4.1 用户消息样式

建议：

- 靠右
- 深色或高饱和主色背景
- 白色文字
- 圆角较大

### 4.2 助手消息样式

建议：

- 靠左
- 浅底卡片
- 保持留白
- 支持 Markdown 渲染

### 4.3 系统状态消息

用于展示：

- 正在更新记忆
- 正在检索
- 正在生成报告

建议：

- 不要太像报错
- 用中性色或品牌辅助色
- 体量小于正常消息

## 5. 右侧状态面板

这是第一版里最值得做的美观与实用点。

它让用户能看到系统“记住了什么”，避免多轮产品看起来只是普通聊天框。

建议分成 3 个卡片。

### 5.1 用户画像卡片

作用：

- 展示当前已提取到的画像
- 让用户快速确认信息是否正确

建议展示字段：

- 学历
- 学校
- 专业
- 目标城市
- 目标岗位
- 技能
- 实习经历

展示方式建议：

- 用小标签和简短文本
- 列表不要太长
- 缺失字段显示“未补充”

### 5.2 当前任务卡片

作用：

- 告诉用户当前咨询进行到哪一步

建议展示：

- 当前主题
- 当前阶段
- 当前目标

例如：

- 当前主题：数据分析求职
- 当前阶段：比较城市机会
- 当前目标：深圳 vs 广州

### 5.3 待补充信息卡片

作用：

- 告诉用户现在还缺什么
- 帮助用户理解为什么系统会追问

建议展示：

- 毕业年份
- 项目经历
- 工作年限

这张卡片最好很轻，不要像“报错提醒”，更像“下一步建议”。

## 6. 底部输入区与快捷操作区

这里不能只有按钮，必须包含用户输入框，因为它才是多轮对话的主入口。

### 6.1 用户输入框

这是页面必需组件。

作用：

- 承接多轮对话主输入
- 让用户持续补充背景、修正信息、继续追问
- 作为页面最常用的交互入口

建议交互：

- 默认单行，内容变长时自动扩成多行
- `Enter` 发送
- `Shift + Enter` 换行
- 发送中临时禁用
- 支持 placeholder 引导用户输入

建议 placeholder：

- `继续补充你的背景、目标岗位、目标城市，或直接追问下一步问题...`

视觉建议：

- 宽度足够大
- 圆角明显
- 轻边框
- 聚焦时有清晰但克制的高亮

### 6.2 发送按钮

作用：

- 发送当前输入框内容

说明：

- 它是输入框的配套按钮，不能缺失
- 如果输入为空，应置灰
- 视觉上应清晰，但不必压过“生成报告”按钮

### 6.3 `继续分析`

作用：

- 用当前记忆继续回答用户新问题
- 本质上是普通聊天输入的补充动作

说明：

- 如果用户已经有明确输入，这个按钮可以不单独存在
- 如果保留，建议它是一个弱按钮

### 6.4 `生成报告`

这是最重要的按钮。

作用：

- 基于当前会话累计记忆
- 调用完整 `Researcher -> Analyst -> Advisor`
- 输出正式 Markdown 报告

这是页面的主按钮，视觉上应最突出。

### 6.5 `导出 Markdown`

作用：

- 导出当前已生成报告
- 方便用户保存分析结果

说明：

- 如果当前还没有报告，按钮应置灰

### 6.6 `重置会话`

作用：

- 清空当前对话上下文
- 重新开始一个新会话

说明：

- 应有二次确认
- 这是危险操作，颜色上应区别于主按钮

## 7. 第一版按钮数量建议

第一版不宜做太多按钮，否则会像操作台而不是产品。

建议保留：

1. `发送`
2. `生成报告`
3. `导出 Markdown`
4. `重置会话`

可选保留：

5. `继续分析`

也就是说，理想数量是 4 到 5 个按钮。

## 8. 对话中的建议型快捷按钮

除了固定按钮，还建议在某些回复后增加上下文快捷动作。

例如系统回答后可附带：

- `继续比较城市`
- `分析目标岗位`
- `生成完整报告`

这种按钮的好处是：

- 减少用户输入负担
- 提高交互流畅度
- 看起来更像一个“有流程感的产品”

但第一版建议每轮最多出现 2 到 3 个，避免太满。

## 9. 最小用户流程

建议第一版前端支持下面这条路径：

1. 用户进入页面
2. 顶部看到 `JobOracle` 品牌和说明
3. 在底部输入框直接输入问题
4. 左右看到对话和当前画像/任务状态
5. 系统在对话中给出短回复和必要追问
6. 用户补充信息若干轮
7. 用户点击 `生成报告`
8. 页面出现报告结果卡片
9. 用户点击 `导出 Markdown`

这条链路跑通，前端就已经足够可用。

## 10. 美观方向

虽然这是临时前端，但不要做成默认 demo 风格。

建议的视觉方向：

- 专业、克制、偏咨询工具感
- 不要过度炫技
- 不要太像后台管理系统

### 10.1 色彩建议

建议使用：

- 主色：深蓝或墨青
- 辅助色：青绿或暖橙
- 背景：暖灰白或浅冷灰
- 危险色：柔和红

不建议：

- 默认紫色系
- 过亮纯蓝
- 大面积纯黑

推荐风格倾向：

- 主色：`#16324F` 一类的深海军蓝
- 辅助强调：`#2E8B7D` 一类的青绿色
- 背景：`#F6F4EF` 或 `#F7F8FA`

### 10.2 字体建议

如果 `Chainlit` 可方便覆写样式，建议避免太普通的默认字体。

优先方向：

- 中文正文：清晰、现代、偏中性
- 标题：更有识别度，但不要花哨

如果前端样式能力有限，至少保证：

- 标题字重明显
- 正文舒适
- 行距不要太挤

### 10.3 卡片风格建议

建议：

- 圆角中等偏大
- 轻阴影
- 卡片之间留白明显
- 不要厚边框

整体感觉应偏“咨询工作台”，而不是“开发调试面板”。

### 10.4 动效建议

第一版只做轻动效：

- 新消息淡入
- 报告生成时 loading 过渡
- 按钮 hover 轻微反馈

不要做复杂动画。

## 11. 第一版推荐的 UI 组成

如果只做最小可用又尽量美观，建议包含：

1. 品牌头部
2. 聊天消息流
3. 底部输入框
4. 发送按钮
5. 用户画像卡片
6. 当前任务卡片
7. 待补充信息卡片
8. `生成报告` 主按钮
9. `导出 Markdown` 次按钮
10. `重置会话` 危险按钮

这是当前性价比最高的一版。

## 12. 组件优先级

### 必做

- 对话区
- 用户输入框
- 发送按钮
- 用户画像卡片
- `生成报告`
- `重置会话`

### 强烈建议做

- 当前任务卡片
- 待补充信息卡片
- `导出 Markdown`

### 可后补

- 建议型快捷按钮
- 更丰富的状态提示
- 会话列表

## 13. 当前结论

`Chainlit` 第一版不需要大而全，但必须有“产品感”。

最推荐的结构是：

- 中间聊天
- 右侧状态
- 底部输入框
- 底部 4 到 5 个关键按钮

最关键的按钮是：

1. `发送`
2. `生成报告`
3. `导出 Markdown`
4. `重置会话`

最关键的展示区是：

1. 对话区
2. 输入区
3. 用户画像区
4. 当前任务与待补充信息区

只要把这几部分做得整洁、留白充足、颜色克制，哪怕是临时前端，也可以做到比较美观。
