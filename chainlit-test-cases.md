# JobOracle Chainlit 测试用例

这份文档用于测试当前 `Chainlit` 临时前端的核心能力：

- 多轮对话
- 用户画像提取与更新
- 右侧状态面板
- LLM 聊天回复
- 流式输出
- 显式生成报告
- 导出 Markdown
- 重置会话

## 1. 启动方式

在项目根目录执行：

```bash
uv run chainlit run src/JobOracle/ui/chainlit_app.py
```

或：

```bash
.venv/bin/chainlit run src/JobOracle/ui/chainlit_app.py
```

浏览器打开：

```text
http://127.0.0.1:8000
```

## 2. 测试重点

测试时重点观察这几类行为：

- 回复是否流式输出
- 聊天回复是否由 LLM 主导，而不是固定模板
- 右侧状态面板是否会更新
- 用户画像是否逐步累积
- 生成报告后是否仍留在同一个会话
- `导出 Markdown` 和 `重置会话` 是否正常

## 3. 主测试用例

建议按下面顺序逐条发送。

### 第 1 轮：基础背景输入

发送：

```text
我是统计学本科，想在深圳找数据分析工作，会 SQL 和 Python。
```

预期：

- 回复应是自然顾问式短回复
- 如果 LLM 可用，应看到流式输出
- 右侧“用户画像”应出现：
  - 学历：本科
  - 专业：统计学
  - 目标城市：深圳
  - 目标岗位：数据分析
  - 技能：SQL、Python
- 右侧应显示：
  - 当前回复方式：LLM 或 fallback
  - 聊天检索：已启用或未启用

### 第 2 轮：补充经历

发送：

```text
我还有一段电商运营实习经历，也做过一些用户增长分析。
```

预期：

- 回复应知道你还在同一个求职话题里
- 右侧“用户画像”应新增：
  - 实习经历
- 不应丢失前面已提取的学历、岗位、城市、技能

### 第 3 轮：短追问

发送：

```text
那广州呢？
```

预期：

- 回复应自动理解为“广州和之前的深圳做比较”
- 不应把“广州”当成全新无上下文问题
- 如果记忆更新成功，右侧目标城市可能变成：
  - 深圳、广州
- 如果触发检索，右侧应显示聊天检索已启用

### 第 4 轮：公司层级判断

发送：

```text
我更适合先投中厂还是小厂？
```

预期：

- 回复应给出明确倾向
- 应结合前文背景，而不是泛泛而谈
- 应给出下一步投递建议

### 第 5 轮：准备策略

发送：

```text
那我现在应该先补项目，还是先投简历试水？
```

预期：

- 回复应更偏策略建议
- 不应切换成报告风格
- 仍然应延续当前会话上下文

### 第 6 轮：调整岗位方向

发送：

```text
其实我也可以接受商业分析，不一定只做纯数据分析。
```

预期：

- 右侧“目标岗位”应更新或扩展
- 回复应调整建议，不应只围绕最初的“数据分析”

### 第 7 轮：补充结构化字段

发送：

```text
我 26 年毕业，本科在江西财经大学。
```

预期：

- 右侧“用户画像”应新增：
  - 学校：江西财经大学
  - 毕业年份：2026 或 26 年相关表达的标准化结果
- 前面已有字段仍应保留

### 第 8 轮：生成报告

点击按钮：

```text
生成报告
```

或发送：

```text
帮我生成报告
```

预期：

- 页面先显示“正在根据当前会话内容生成报告”
- 报告生成后仍停留在当前会话
- 不应自动新开会话
- 报告内容应综合前面多轮上下文

### 第 9 轮：导出 Markdown

点击：

```text
导出 Markdown
```

预期：

- 能看到最近一次生成的报告内容
- 如果报告保存成功，应显示保存路径

### 第 10 轮：重置会话

点击：

```text
重置会话
```

然后发送：

```text
我想找算法岗。
```

预期：

- 右侧画像不应再保留前一位“统计学本科 / 数据分析 / 深圳”的信息
- 新会话应从空白状态开始

## 4. 边界测试

### 用例 A：城市扩展

发送：

```text
我不一定去深圳，也可以去杭州或者广州。
```

预期：

- 目标城市应支持多值
- 不应只保留一个城市

### 用例 B：目标修正

发送：

```text
我之前说的数据分析不用当成唯一方向，我更想试商业分析或者增长分析。
```

预期：

- 系统应理解这是修正/扩展，不只是无脑追加

### 用例 C：学历表达模糊

发送：

```text
我现在也不确定是不是一定读研，所以先按本科来考虑吧。
```

预期：

- 画像提取应尽量谨慎
- 不应轻易把学历误改成硕士

## 5. 异常观察点

如果测试时出现以下现象，建议记录下来：

- 回复不是流式，而是整段突然出现
- 右侧状态面板不更新
- 画像字段丢失或被错误覆盖
- 生成报告后自动新开会话
- 回复重新退回明显模板化
- 页面显示未连接到服务器

记录时尽量同时保留：

- 你的输入原文
- 系统回复
- 右侧状态变化
- 终端报错或前端异常提示
